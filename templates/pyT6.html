6. Booleans and Further Logic

6.1 Booleans

Booleans can be thought of as variables that store two cases: true and false. It is returned by comparison operators, such as the if statements discussed above.

For example:
Is 4 less than 5? We know this is the case, in other words, it is true. We are able to compare such things in Python and it will return either true/false. This can be seen in the example below:

>>> 4 < 5
True

We can see that the statement is true.
Conversely, try the opposite:

>>> 5 < 4
False

We can see that Python outputs false.
Python uses this logic in if statements. If we go back to our soup flowchart (from the previous article), we can view yes/no as the boolean outputs of True/False that the if statement outputs.

In if soup_time >= 10 we can say that soup_time >= 10 will either be True or False.

If the result of this comparison is true, we will stop cooking. In this way, we can see how Python uses boolean operations to make decisions in our comparison statements.

6.2 Logical Operators

Sometimes you want multiple conditions to be met before something happens in your code. You can do this using and. It checks whether both conditions are true and if either isn't, then it will output false.

>>> True and True
True
>>> True and False
False

Let's apply this to our previous example. We only want to stop cooking our soup after 10 minutes but we also have to make sure we don't cook it for more than 20 minutes. How do we apply this?

soup_time = int(input('How long have you been cooking the soup?'))
if soup_time >= 10 and soup_time < 20:
  print('The soup is ready')
elif soup_time < 10:
  print('Keep cooking')
else:
  print('You've cooked it for too long!')

Additionally, we have two other operators for booleans: or and not. or checks if either condition is true. whereas not gives you the opposite of the boolean.

>>> True or False
True
>>> True or True
True
>>> False or False
False
>>> not True
False
>>> not False
True

These are all very useful when you have very specific criteria in your if statements. Make sure you're comfortable with them!
